### Added

- **Resilient publish** — `createResilientPublisher<T>()` with retry, exponential / linear / constant backoff, and full / equal / decorrelated jitter. Rejects with a typed `ResilientPubSubError` on permanent failure; never silently drops an event.
- **Resilient subscribe** — `createResilientSubscriber<T>()` with correct ack/nack lifecycle: handler throws → nack (Pub/Sub redelivers), handler returns → ack (done). Includes observable hooks (`onAck`, `onNack`, `onPoison`, `onError`) and graceful `stop()` drain.
- **Symmetric typed envelope** — `Envelope<T>`: publish `{ body, headers }`, receive `{ body, headers, meta }`. Pluggable `Serializer<T>` interface with a built-in `JsonSerializer<T>`.
- **W3C context propagation** — `injectContext` / `extractContext` carry `traceparent` / `tracestate` and allowlisted business headers across the message hop, zero runtime dependencies.
- **Native dead-letter support** — `buildDeadLetterPolicy` / `withDeadLetter` builder (opt-in). `deliveryAttempt` surfaced on `msg.meta`. `onPoison` hook for SerializationError routing.
- **Safe error surface** — `ResilientPubSubError` with explicit `kind`, `classification`, `retryable`, and `toJSON()` that never leaks secrets, body, cause, or raw attributes. `isResilientPubSubError` brand guard works across module/realm boundaries. `SerializationError` fixed-semantics subclass (`classification = 'poison'`).
- **Core algorithms** — `calculateBackoff`, `applyJitter`, `classify`, `isRetryable`; zero runtime dependencies; tree-shakeable.
- **Config module** — `RESILIENT_PUBSUB_*` environment-variable convention with programmatic override and safe defaults.
- **TypeScript-first** — full `.d.ts` generated by tsup; dual ESM + CJS build; conditional exports map; `sideEffects: false`.
- **Governance** — `CONTRIBUTING.md`, `SECURITY.md` (private vulnerability reporting), `.github/dependabot.yml` (npm + github-actions, weekly).
